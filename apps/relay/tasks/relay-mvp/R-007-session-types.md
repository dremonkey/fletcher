# R-007: Session Types (TypeScript Schemas for State)

**Depends On:** R-001  
**Blocks:** R-008, R-010  
**Effort:** 1 hour  

## Objective
Define TypeScript types for session state management.

## Reference
See `docs/gateway-api-contract.md` section "Session Persistence"

## Key File
- `src/session/types.ts`

## Types to Define
```typescript
export interface Session {
  sessionId: string;           // UUID generated by relay
  sessionKey: string;           // OpenClaw session key (derived from user/room)
  openclawSessionId?: string;   // OpenClaw internal session ID (from response)
  roomName: string;             // LiveKit room name
  userId?: string;              // Participant identity (if available)
  messages: ChatMessage[];      // Conversation history
  state: SessionState;
  createdAt: number;            // Unix timestamp
  lastActivity: number;         // Unix timestamp
}

export enum SessionState {
  Active = 'active',
  Idle = 'idle',
  Completed = 'completed',
  Error = 'error',
}

export interface ChatMessage {
  role: 'user' | 'assistant' | 'system';
  content: string;
  timestamp: number;
}
```

## Acceptance Criteria
✅ All session fields typed  
✅ SessionState enum defined  
✅ ChatMessage interface matches OpenClaw format
