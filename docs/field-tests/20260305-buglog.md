## Detailed Entries

### BUG-001: Gemini TTS rate limiting — 429 quota exhausted (HIGH)

**First seen:** 15:34:22 UTC
**Frequency:** Recurring bursts (~15:34, ~16:41, ~17:05 UTC)
**Impact:** Agent speech output fails; client shows rapid-fire "Pipeline Error" artifacts. User hears nothing from the agent.

**Log excerpt:**
```
[15:34:22.970] ERROR Pipeline error
  source: "Pipeline"
  message: "Gemini TTS: Connection error - 429 ... You exceeded your current quota ...
    Quota exceeded for metric: generativelanguage.googleapis.com/generate_requests_per_model,
    limit: 10, model: gemini-2.5-flash-tts
    Please retry in 37s."
```

**Analysis:** The `gemini-2.5-flash-tts` model has a hard limit of 10 requests per minute per project. The TTS recovery loop retries immediately on failure, which burns through the remaining quota in seconds and creates a cascade of 429s. Each failed TTS attempt emits a "Pipeline Error" artifact to the client — the client received 1369 such artifacts in the 08:58–09:04 local time window alone.

**Root cause:** Free-tier Gemini API rate limit (10 RPM) combined with aggressive TTS retry behavior that doesn't respect the `retryDelay` hint in the 429 response.

**Fix applied:** None yet. Needs either:
1. Rate-limit-aware backoff in TTS retry loop (respect `retryDelay` from 429 response)
2. Upgrade Gemini API tier for higher RPM
3. Suppress "Pipeline Error" artifact emission during TTS recovery retries

### BUG-003: Pipeline Error artifact flood on client (HIGH)

**First seen:** 16:58 UTC (08:58 local)
**Frequency:** Continuous for ~6 minutes (08:58–09:04 local)
**Impact:** Client received 1369 "Pipeline Error" artifact messages. This is a UX issue — even if TTS recovery is retrying, the client should not be spammed with error artifacts.

**Log excerpt (client):**
```
03-05 08:58:30.079 I/flutter: [Ganglia] Artifact: Pipeline Error
03-05 08:58:30.349 I/flutter: [Ganglia] Artifact: Pipeline Error
03-05 08:58:30.780 I/flutter: [Ganglia] Artifact: Pipeline Error
... (1369 total over ~6 minutes)
03-05 09:04:27.461 I/flutter: [Ganglia] Artifact: Pipeline Error
```

**Analysis:** Each TTS retry failure emits a pipeline error event that gets forwarded to the client as a data channel artifact. The client has no deduplication or rate-limiting for these artifacts. This is related to BUG-001 — the retry storm causes an artifact storm.

**Root cause:** The existing 5s TTS debounce only matched `err.type === 'tts_error'`, but Gemini 429 errors arrived as generic pipeline errors (no `tts_error` type), bypassing the debounce entirely.

**Fix applied:** 17:15 UTC
- Changed debounce from TTS-only to all error artifacts
- Increased debounce window from 5s to 60s (1 per minute max)
- Added `message.includes('TTS')` heuristic to detect TTS errors regardless of `err.type`
- Also fixed pre-existing unterminated string literal in `llm.ts:242`
- File: `apps/voice-agent/src/agent.ts` (lines 298–351)

### BUG-004: Voice agent high memory usage (HIGH)

**First seen:** 14:35:01 UTC (pre-restart session)
**Frequency:** Continuous warnings every 5 seconds
**Impact:** Voice agent process consumed 7.4 GB of memory (threshold: 500 MB). Could lead to OOM kills or degraded performance.

**Log excerpt:**
```
[15:34:27.464] WARN process memory usage is high
  memoryUsageMB: 7406.52734375
  memoryWarnMB: 500
```

After restart at 16:32 UTC, memory stabilized around ~780 MB (still above 500 MB threshold but not catastrophic).

**Analysis:** The voice agent has a memory leak. The pre-restart session ran for an extended period and accumulated 7.4 GB. Post-restart, memory climbed to ~780 MB in about 30 minutes. The leak likely comes from retained TTS/STT buffers or unreleased streams during error recovery.

**Root cause:** Primary: `_AudioOut.audio: Array<AudioFrame>` in SDK `generation.ts:753` accumulates every TTS audio frame (~96 KB/sec) and is never read or cleared. The array is retained via closure chains in `SpeechHandle._tasks[]`. Secondary: OpenTelemetry spans with `endOnExit: false` never ended on abort. Minor: `TranscriptManager.knownStreamIds` grows forever. Amplified by TTS 429 retry storms creating thousands of short-lived streams.

**Task:** [`tasks/02-livekit-agent/017-voice-agent-memory-leak.md`](../../tasks/02-livekit-agent/017-voice-agent-memory-leak.md)
