**First seen:** 08:52:47 UTC (memory warnings begin at 513 MB, 15 min after startup)
**Frequency:** Continuous — memory grew monotonically with no voice interactions
**Impact:** Agent became unresponsive, then dropped out of room entirely at 10:51.

**Analysis:** Yesterday's RCA (task 017) identified `_AudioOut.audio` array accumulation as the primary leak during active TTS sessions. Today's data reveals a **separate idle-session leak**: 836K GoogleAuth-related objects created in 53 minutes with no TTS calls. This points to a hot loop in the Google TTS plugin or its auth library that runs continuously even when no synthesis is requested — possibly a token refresh timer, health check, or initialization loop.

**Root cause:** Under investigation. The GoogleAuth churn is a new vector not covered by task 017's existing fixes.

### BUG-002: Agent DTLS timeout — dropped from room (HIGH)

**First seen:** 10:51:18 UTC
**Frequency:** Once (fatal for the session)
**Impact:** Agent silently left the room. Client remained connected to LiveKit for ~5 more hours (until 15:45) but had no agent to talk to. No error shown to user.

**Log excerpt:**
```
livekit-1 | 2026-03-06T10:51:18.959Z  WARN  error reading data channel
  participant: "agent-AJ_sXTgEV4ERXQh"
  label: "_lossy"
  error: "dtls timeout: read/write timeout: context deadline exceeded"
```

**Analysis:** The agent process was so memory-starved (2.6 GB RSS) that it couldn't maintain WebRTC connections. DTLS timeouts killed both data channels. The agent container stayed running (Docker reports 0 restarts) but the process was zombie — no further logs emitted until a new job at 16:12.

The client reconnected at 13:58 (ICE reconnect) and again at 15:45, but LiveKit showed `numParticipants: 0` — confirming the agent was gone from the room.

**Root cause:** Consequence of BUG-001 memory leak.

### BUG-003: Turn detector inference slower than realtime (MEDIUM)

**First seen:** 09:30:23 UTC
**Frequency:** 12,989 warnings between 09:30 and 10:50
**Impact:** End-of-utterance detection lagged behind audio input — agent unable to detect speech turns.

**Log excerpt:**
```
voice-agent-1 | [09:30:23.572] WARN (201): inference is slower than realtime
  delay: 201
```

**Analysis:** The ONNX turn detector processes every mic audio frame. Under memory pressure, inference throughput dropped below realtime, making the agent effectively deaf. This is a symptom of BUG-001, not a separate issue.

**Root cause:** Memory pressure from BUG-001.
