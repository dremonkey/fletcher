# T29.07 — Latency Measurement

**Status:** Not started
**EPIC:** [29-relay-claude-code](./EPIC.md)
**PRD Requirement:** NFR1
**Depends on:** T29.04

## Objective
Measure first-token latency for the Claude Code backend and ensure it meets performance targets relative to the OpenClaw baseline.

## Details
- Measure first-token latency: time from `session/prompt` send to first `agent_message_chunk` received
- Collect p50 and p95 latency across a representative sample of prompts
- Compare results against the OpenClaw baseline
- Target: p95 < 3 seconds, no regression > 20% vs baseline

## Acceptance Criteria
- [ ] First-token latency measured for p50 and p95
- [ ] p95 latency is under 3 seconds
- [ ] No latency regression greater than 20% compared to OpenClaw baseline
- [ ] Results documented with methodology and sample size

## Notes
- Use consistent prompt complexity for fair comparison
- Test under both cold-start and warm conditions
- Record environment details (machine specs, network) alongside measurements
