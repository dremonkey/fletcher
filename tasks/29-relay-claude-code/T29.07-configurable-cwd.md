# T29.07 — Configurable Working Directory (cwd)

**Status:** Not started
**EPIC:** [29-relay-claude-code](./EPIC.md)
**Depends on:** T29.03

## Objective

Allow the mobile client to select which project directory the ACP agent operates in, using a relay-managed allowlist of presets and glob patterns.

## Background

The `cwd` passed in `session/new` controls:
- Where agent tools (Read, Write, Bash, Edit) execute
- Which `CLAUDE.md` project instructions are loaded
- Where session history is stored (`~/.claude/projects/<sanitized-cwd>/`)

Currently hardcoded to `process.cwd()` in 5 locations. This task makes it configurable via presets + allowlist, with the mobile client selecting a workspace at bind time.

## Design

### Config: `apps/relay/workspace-presets.json`

```json
{
  "default": "fletcher",
  "presets": {
    "fletcher": "/home/ahanyu/code/fletcher",
    "openclaw": "/home/ahanyu/code/openclaw"
  },
  "allowPatterns": [
    "/home/ahanyu/code/*"
  ]
}
```

- `default` — preset name when mobile doesn't specify cwd
- `presets` — named mappings to absolute paths
- `allowPatterns` — glob patterns for arbitrary paths (`*` = one segment, `**` = any depth)
- Fallback chain: config default → `ACP_CWD` env var → `process.cwd()`

### Resolution logic

Mobile sends optional `cwd` in `session/bind` params. Relay resolves:

1. No cwd → use default
2. Matches preset name → use preset path
3. Absolute path matching allowlist → use it
4. Otherwise → reject with `-32602` error

Path normalized via `path.resolve()` before matching (prevents traversal). Validated to exist and be a directory.

### Protocol change: `session/bind`

```json
// Request (cwd is optional)
{ "method": "session/bind", "params": { "sessionKey": "...", "cwd": "fletcher" } }

// Response includes resolved workspace
{ "result": { "sessionKey": "...", "bound": true, "cwd": "/home/ahanyu/code/fletcher" } }
```

### Discovery: `GET /relay/presets`

```json
{ "presets": { "fletcher": "/home/ahanyu/code/fletcher" }, "default": "fletcher" }
```

Mobile fetches alongside existing health checks. Workspace picker UI deferred.
