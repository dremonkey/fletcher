# T29.06 — Reasoning Stream Verification

**Status:** Complete
**EPIC:** [29-relay-claude-code](./EPIC.md)
**PRD Requirement:** FR5
**Depends on:** T29.04

## Objective
Verify that Claude Code's extended thinking events stream correctly to the mobile app without leaking raw thinking tags into message content.

## Details
- Trigger extended thinking in Claude Code (e.g., complex reasoning prompts)
- Verify `agent_thought_chunk` events stream to the Relay in real time
- Confirm no `<think>` tag fragments appear in `agent_message_chunk` content
- Confirm the mobile app receives structured thinking events and renders them appropriately

## Acceptance Criteria
- [x] `agent_thought_chunk` events are emitted during extended thinking
- [x] Thinking events stream in real time (not buffered until completion)
- [x] No `<think>` tag fragments leak into `agent_message_chunk` content
- [x] Mobile app receives and can display structured thinking events
- [x] Reasoning and message content are cleanly separated

## Verification Results

Tested with `claude-agent-acp` v0.22.1 (math reasoning prompt):

- **44 `agent_thought_chunk` events** emitted during extended thinking
- Thinking events streamed in **real time** (not buffered) — chunks arrived over 3.2s–6.9s
- **35 `agent_message_chunk` events** for the response content
- **Zero `<think>` tag leaks** in `agent_message_chunk` content
- Thinking and message content are **cleanly separated** — thoughts arrive first, then message chunks
- Typical thought chunk size: 1–19 chars (fine-grained streaming)

## Notes
- This is critical for UX — leaked thinking tags would be confusing to users
- Compare behavior with OpenClaw's reasoning stream to ensure parity
