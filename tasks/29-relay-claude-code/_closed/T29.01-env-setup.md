## Objective
Install and configure an ACP-compatible Claude backend so the Relay can spawn it as a subprocess.

## Key Finding
Claude Code CLI has **no native ACP support**. There is no `--acp` flag. GitHub issue #6686 requesting one was closed without implementation.

## Solution
Using `@zed-industries/claude-agent-acp` v0.22.1 — a Zed-maintained ACP adapter that wraps the Claude Agent SDK (`@anthropic-ai/claude-agent-sdk`). The adapter is self-contained: it embeds the Claude CLI internally and exposes a pure stdio ACP (JSON-RPC 2.0) server.

## Details
- Installed globally via npm: `npm install -g @zed-industries/claude-agent-acp`
- Binary location: `/home/ahanyu/.local/bin/claude-agent-acp`
- Authentication: `ANTHROPIC_API_KEY` env var (NOT `CLAUDE_CODE_OAUTH_TOKEN`)
- The adapter is self-contained — embeds Claude CLI via `@anthropic-ai/claude-agent-sdk`
- No additional flags needed — the binary is a pure stdio ACP server

## Acceptance Criteria
- [x] `claude-agent-acp` binary installed and on `$PATH`
- [x] Authentication works via `ANTHROPIC_API_KEY` env var
- [x] Binary spawns and accepts JSON-RPC on stdin
- [x] CI/dev environment can resolve `claude-agent-acp`

## Notes

- The Zed adapter is a third-party dependency (`@zed-industries/claude-agent-acp`)
- For production/CI, `ANTHROPIC_API_KEY` must be available in the process environment
