# T29.03 — Session Config Compatibility

**Status:** Complete [x]
**EPIC:** [29-relay-claude-code](./EPIC.md)
**PRD Requirement:** FR3
**Depends on:** T29.02

## Objective
Extend the ACP session config to support Claude Code and make the Relay bridge select the correct config section based on the connected agent.

## Changes Made
- Removed hardcoded `--session` from subprocess args — session key now passed via `_meta.session_key` in `session/new`
- Config selection uses `agentInfo.name` from `initialize` response, with `ACP_COMMAND` env var as fallback
- Added `@zed-industries/claude-agent-acp` section to `acp-session-config.json`
- Added `AgentInfo` type to `@fletcher/acp-client`
- All 27 relay-bridge tests pass

## Acceptance Criteria
- [x] `acp-session-config.json` contains a `"@zed-industries/claude-agent-acp"` section
- [x] `relay-bridge.ts` dynamically selects config based on `agentInfo.name`
- [x] Unknown config options are handled gracefully (no crash, logged warning)
- [x] Existing OpenClaw sessions are unaffected

## Notes
- Config selection: `agentInfo.name` match first, then `ACP_COMMAND` fallback, then default section
- Key files: `apps/relay/acp-session-config.json`, `apps/relay/src/bridge/relay-bridge.ts`, `packages/acp-client/src/types.ts`
