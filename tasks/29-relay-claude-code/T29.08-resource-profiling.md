# T29.08 — Resource Profiling

**Status:** Not started
**EPIC:** [29-relay-claude-code](./EPIC.md)
**PRD Requirement:** NFR3
**Depends on:** T29.02

## Objective
Profile the Claude Code subprocess resource usage and assess its impact on single-machine deployment.

## Details
- Measure Claude Code subprocess startup time (spawn to ready)
- Measure idle RSS (resident set size) when subprocess is waiting
- Measure active RSS during prompt processing and tool execution
- Compare all metrics against the OpenClaw baseline
- Assess whether resource usage is acceptable for single-machine deployment

## Acceptance Criteria
- [ ] Subprocess startup time measured and documented
- [ ] Idle RSS measured and documented
- [ ] Active RSS measured and documented
- [ ] Comparison against OpenClaw baseline recorded
- [ ] Assessment of single-machine deployment impact provided

## Notes
- If RSS is significantly higher than OpenClaw, consider subprocess pooling or on-demand spawning
- Monitor for memory leaks over extended sessions
