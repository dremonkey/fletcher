# T29.01 — Environment Setup

**Status:** Not started
**EPIC:** [29-relay-claude-code](./EPIC.md)
**PRD Requirement:** FR1
**Depends on:** None

## Objective
Install and configure the Claude Code CLI so that the Relay backend can spawn it as an ACP subprocess.

## Details
- Install `claude` CLI via the `@anthropic-ai/claude-code` npm package or standalone binary
- Verify authentication: `CLAUDE_CODE_OAUTH_TOKEN` (already set in `~/.zshrc`) is sufficient — no separate `ANTHROPIC_API_KEY` needed
- Verify `claude` is on `$PATH` and executable from the Relay process context
- Confirm the installed version supports ACP mode

## Acceptance Criteria
- [ ] `claude --version` succeeds in the Relay runtime environment
- [ ] Authentication works via `CLAUDE_CODE_OAUTH_TOKEN` (inherited from shell environment)
- [ ] `claude` binary is on `$PATH` and executable
- [ ] CI/dev environment can also resolve `claude`

## Notes
- Check `flake.nix` for dev environment setup — may need to add `claude` to nix packages
- Auth uses OAuth token (`CLAUDE_CODE_OAUTH_TOKEN`) from `~/.zshrc`, not a separate API key
- For production/CI, ensure the token is available in the process environment (secrets manager or `.env`)
