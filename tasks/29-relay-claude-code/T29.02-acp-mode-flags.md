# T29.02 — Determine ACP Mode Flags

**Status:** Not started
**EPIC:** [29-relay-claude-code](./EPIC.md)
**PRD Requirement:** FR2
**Depends on:** T29.01

## Objective
Identify the exact CLI flags required to launch Claude Code in ACP stdio mode and validate that the subprocess accepts JSON-RPC on stdin.

## Details
- Run `claude --help` or consult official docs to find the exact flags for ACP stdio mode
- Test candidate flag combinations (e.g., `claude code --acp`, `claude --acp`)
- Validate that the subprocess spawns cleanly with those flags
- Confirm the subprocess accepts JSON-RPC messages on stdin and responds on stdout

## Acceptance Criteria
- [ ] Exact ACP mode flags are documented
- [ ] Subprocess spawns without error using identified flags
- [ ] Subprocess accepts a JSON-RPC `initialize` request on stdin and returns a valid response on stdout
- [ ] Flags are recorded in the EPIC or architecture doc for downstream tasks

## Notes
- Likely `claude code --acp` or `claude --acp` — needs investigation against latest CLI version
- If flags differ between CLI versions, pin the minimum supported version
