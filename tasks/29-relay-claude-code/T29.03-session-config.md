# T29.03 — Session Config Compatibility

**Status:** Not started
**EPIC:** [29-relay-claude-code](./EPIC.md)
**PRD Requirement:** FR3
**Depends on:** T29.02

## Objective
Extend the ACP session config to support Claude Code and make the Relay bridge select the correct config section based on the connected agent.

## Details
- Add a `"claude-code"` section to `acp-session-config.json` with Claude Code-specific options
- Update `relay-bridge.ts` to select the config section based on `agentInfo.name` from the `initialize` response
- Handle the case where Claude Code rejects unknown config options (graceful fallback)
- Preserve backward compatibility with the existing OpenClaw config section

## Acceptance Criteria
- [ ] `acp-session-config.json` contains a `"claude-code"` section
- [ ] `relay-bridge.ts` dynamically selects config based on `agentInfo.name`
- [ ] Unknown config options are handled gracefully (no crash, logged warning)
- [ ] Existing OpenClaw sessions are unaffected
