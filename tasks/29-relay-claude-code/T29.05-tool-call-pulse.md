# T29.05 — Tool-Call Pulse Verification

**Status:** Not started
**EPIC:** [29-relay-claude-code](./EPIC.md)
**PRD Requirement:** FR4
**Depends on:** T29.04

## Objective
Verify that tool-call lifecycle events from Claude Code are correctly relayed to the mobile app and keep the connection alive.

## Details
- Run multi-tool conversations through the Claude Code backend
- Verify `tool_call` event fires on invocation and `tool_call_update` fires during execution
- Confirm the mobile app receives these events and resets its 30-second timeout timer
- Verify at least one `session/update` is emitted per 5 seconds during active tool use

## Acceptance Criteria
- [ ] `tool_call` event is emitted when Claude Code invokes a tool
- [ ] `tool_call_update` events stream during tool execution
- [ ] Mobile app receives events and resets 30s timer (no premature timeout)
- [ ] At least one `session/update` per 5s during active tool use
- [ ] Multi-tool conversations complete end-to-end without dropped events

## Notes
- Key files: `apps/relay/src/bridge/relay-bridge.ts`, `apps/mobile/lib/services/relay/relay_chat_service.dart`
- Test with tools that have both short and long execution times
