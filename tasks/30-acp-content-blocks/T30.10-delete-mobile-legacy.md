# Task 30.10: Delete Mobile Legacy Artifacts

**Epic:** 30 — ACP Content Blocks
**Status:** [ ]
**Depends on:** T30.02, T30.03, T30.08, T30.09
**Blocks:** T30.11

## Goal

Delete all legacy artifact infrastructure from the mobile app: `ArtifactEvent`, `ArtifactType`, `ganglia-events` content handlers (artifact, status, text_message), `showArtifactDrawer()`, and deprecated `RoomNameGenerator.generate()`. After this task, mobile receives all content exclusively through the `acp` topic and renders via `RendererRegistry`.

## Context

By the time this task runs, all prerequisites are complete:
- T30.02: Status comes from ACP `tool_call` (not ganglia-events `StatusEvent`)
- T30.03: Text routes through relay (not ganglia-events `text_message`)
- T30.08: AcpUpdateParser emits `ContentBlock` for all content types
- T30.09: Renderers accept `ContentBlock` input via `RendererRegistry`

The ganglia-events handlers that remain AFTER this deletion:
- `pondering` → ThinkingSpinner (voice control)
- `session_hold` → hold mode (voice control)
- `tts-mode` → TTS toggle (voice control)
- `agent_transcript` / `user_transcript` → subtitle display (voice control)
- `pipeline_info` → diagnostics (voice control)

### 2. Delete ganglia-events content handlers (`apps/mobile/lib/services/livekit_service.dart`)

In `_processGangliaEvent()` (lines ~1124-1221), remove cases for:

### 7. Verify build
