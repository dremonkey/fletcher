# Task 30.11: Gut Ganglia

**Epic:** 30 — ACP Content Blocks
**Status:** [ ]
**Depends on:** T30.10
**Blocks:** none

## Goal

Delete the artifact pipeline and Nanoclaw backend from `livekit-agent-ganglia`, leaving a focused package: `RelayLLM` + `SessionKey` routing + slim factory + types + public API. Also remove `EventInterceptor` usage from the voice agent.

## Context

After T30.10, mobile no longer consumes Ganglia artifacts or status events. The voice agent's `EventInterceptor` still generates them, but nobody listens. Safe to delete.

**What gets deleted:**

| File | Component | Why dead |
|---|---|---|
| `tool-interceptor.ts` | `ToolInterceptor` | Relay provides tool-call content natively |
| `tool-interceptor.spec.ts` | Tests | Dead with source |
| `event-interceptor.ts` | `EventInterceptor` | Relay provides events natively |
| `events.ts` | Custom artifact types | Replaced by ACP ContentBlock |
| `events.spec.ts` | Tests | Dead with source |
| `nanoclaw.ts` | `NanoclawLLM`, `NanoclawChatStream` | Relay is the only backend |
| `nanoclaw-client.ts` | `NanoclawClient` | Dead with Nanoclaw |
| `nanoclaw-client.spec.ts` | Tests | Dead with source |
| `types/nanoclaw.ts` | Misnamed OpenAI-compat types | Only used by Nanoclaw |

**What gets modified:**

| File | Change |
|---|---|
| `factory.ts` | Remove `nanoclaw` case, remove `BRAIN_TYPE` env alias, default to `relay` |
| `factory.spec.ts` | Remove Nanoclaw factory tests |
| `ganglia-types.ts` | Remove `NanoclawConfig`, `historyMode` |
| `index.ts` | Remove all Nanoclaw exports, ToolInterceptor exports, EventInterceptor exports, artifact type exports |

**What survives:**

| File | Component |
|---|---|
| `relay-llm.ts` | `RelayLLM` — routes via data channel to relay |
| `relay-stream.ts` | `RelayChatStream` — streaming responses |
| `relay-transport.ts` | `DataChannelTransport`, `VOICE_ACP_TOPIC` |
| `session-routing.ts` | `resolveSessionKey`, `SessionKey` |
| `pondering.ts` | Pondering phrase rotation |
| `preload.ts` | Pre-loading utilities |
| `logger.ts` | Logger interface |
| `ganglia-types.ts` | `RelayConfig`, `GangliaConfig` (slimmed) |
| `factory.ts` | `createGangliaFromEnv` (relay-only) |
| `index.ts` | Public API (slimmed) |

**Voice agent changes** (`apps/voice-agent/src/agent.ts`):
- Remove `EventInterceptor` instantiation and usage
- Remove artifact stamping logic (lines ~214-219: auto-stamp segmentId on artifacts)
- Remove `publishEvent()` calls that emit artifact/status events
- Keep: `publishEvent()` for voice-control events (pondering, transcription, hold, bootstrap, pipeline_info)

The `publishEvent()` function itself may stay — it's used for voice-control events. But remove the artifact-specific logic (segmentId stamping).

## Implementation

### 1. Remove EventInterceptor from voice agent (`apps/voice-agent/src/agent.ts`)

- Remove EventInterceptor import and instantiation
- Remove artifact segmentId stamping logic
- Remove any `publishEvent()` calls that emit `artifact` or `status` type events for tool execution
- Keep `publishEvent()` for voice-control events

### 2. Delete artifact pipeline files (`packages/livekit-agent-ganglia/src/`)

Delete files:
- `tool-interceptor.ts` + `tool-interceptor.spec.ts`
- `event-interceptor.ts`
- `events.ts` + `events.spec.ts`

### 3. Delete Nanoclaw backend (`packages/livekit-agent-ganglia/src/`)

Delete files:
- `nanoclaw.ts`
- `nanoclaw-client.ts` + `nanoclaw-client.spec.ts`
- `types/nanoclaw.ts`

### 4. Slim factory (`packages/livekit-agent-ganglia/src/factory.ts`)

- Remove `nanoclaw` case from `createGangliaFromEnv()`
- Remove `BRAIN_TYPE` env var fallback (line ~118-124: keep only `GANGLIA_TYPE`)
- Default remains `relay`
- Remove `import './nanoclaw.js'` (self-registration side-effect)

### 5. Slim types (`packages/livekit-agent-ganglia/src/ganglia-types.ts`)

- Remove `NanoclawConfig` interface
- Remove `historyMode` field
- Keep `RelayConfig`, `GangliaConfig`, `GangliaType` (narrowed to `'relay'`)

### 6. Clean exports (`packages/livekit-agent-ganglia/src/index.ts`)

Remove exports for:
- All artifact types (StatusEvent, ArtifactEvent, ArtifactType, etc.)
- ToolInterceptor and related (ToolCall, ToolResult, ToolExecutor, EventEmitter, etc.)
- EventInterceptor
- NanoclawLLM, NanoclawClient, generateChannelJid, sessionKeyToChannel
- `nanoclaw` namespace export

Keep exports for:
- RelayLLM, RelayConfig, DataChannelTransport, VOICE_ACP_TOPIC
- SessionKey routing (resolveSessionKey, resolveSessionKeySimple, etc.)
- Factory (createGanglia, createGangliaFromEnv, registerGanglia)
- Logger (noopLogger, Logger)
- Pondering, preload
- Slimmed types

### 7. Update factory.spec.ts

Remove Nanoclaw factory tests. Keep relay factory tests.

### 8. Verify builds

- `bun test` in `packages/livekit-agent-ganglia/` passes
- `bun test` in `apps/voice-agent/` passes (if it has tests)
- No import errors in dependent packages
