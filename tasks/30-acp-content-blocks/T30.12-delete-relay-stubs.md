# Task 30.12: Delete Deprecated Relay Stubs

**Epic:** 30 — ACP Content Blocks
**Status:** [ ]
**Depends on:** none
**Blocks:** none

## Goal

Delete the four deprecated stub methods from `BridgeManager`: `scheduleRemoveRoom()`, `cancelPendingTeardown()`, `hasPendingTeardown()`, `getPendingTeardowns()`. These are leftover from the deferred-teardown design that was replaced by immediate teardown (BUG-036). They're dead code that returns fixed values.

## Context

**Location:** `apps/relay/src/bridge/bridge-manager.ts` lines 221-241.

All four methods are stubs:
- `scheduleRemoveRoom()` → calls `removeRoom()` immediately
- `cancelPendingTeardown()` → returns `false`
- `hasPendingTeardown()` → returns `false`
- `getPendingTeardowns()` → returns `[]`

The webhook caller at `apps/relay/src/http/webhook.ts` (lines 77-89) references `scheduleRemoveRoom`. Update to call `removeRoom()` directly.

## Implementation

### 1. Delete stubs (`apps/relay/src/bridge/bridge-manager.ts`)

Remove the four deprecated methods (lines ~221-241).

### 2. Update callers (`apps/relay/src/http/webhook.ts`)

Replace `bridgeManager.scheduleRemoveRoom(roomName)` with `bridgeManager.removeRoom(roomName).catch(...)`.

Check for any other callers via grep for `scheduleRemoveRoom`, `cancelPendingTeardown`, `hasPendingTeardown`, `getPendingTeardowns`.

### 3. Tests

- Verify existing tests don't reference the deleted methods
- `bun test` passes for relay
