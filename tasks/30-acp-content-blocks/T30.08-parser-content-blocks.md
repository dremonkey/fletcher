# Task 30.08: AcpUpdateParser Emits ContentBlock

**Epic:** 30 — ACP Content Blocks
**Status:** [ ]
**Depends on:** T30.05, T30.06
**Blocks:** T30.10, T30.14, T30.15, T30.16

## Goal

Widen `AcpUpdateParser` to emit `ContentBlock` instances for all ACP content types, not just text. Currently the parser returns `AcpNonContentUpdate` for any non-text content. After this task, image, audio, resource, resource_link, diff, and terminal content all produce typed `ContentBlock` objects that flow through to the `RendererRegistry`.

## Context

**Current parser** at `acp_update_parser.dart:313-332`:
```dart
final contentType = content['type'];
if (contentType != 'text') {
  return AcpNonContentUpdate(kind);  // everything non-text is dropped
}
final text = content['text'];
```

**After:** The parser dispatches on `content['type']` and constructs the appropriate `ContentBlock` subclass via `ContentBlock.fromJson(content)`.

**Update types affected:**
- `agent_message_chunk` — `content` field is a single ContentBlock
- `agent_thought_chunk` — `content` field (text only, no change needed)
- `tool_call_update` — `content` field is `ToolCallContent[]` (array of content items)
- `user_message_chunk` — `content` field is a ContentBlock (during session/load replay)

**New parser result types needed:**
- `AcpContentDelta` — carries a `ContentBlock` for non-text content (image, audio, resource, etc.)
- Or widen `AcpTextDelta` to carry a `ContentBlock` instead of raw `String text`

Design decision: widen `AcpTextDelta` → `AcpContentDelta` that carries a `ContentBlock`. Text is just one variant. This avoids a parallel hierarchy.

## Implementation

### 1. Replace AcpTextDelta with AcpContentDelta (`apps/mobile/lib/services/relay/acp_update_parser.dart`)

```dart
class AcpContentDelta extends AcpUpdate {
  final ContentBlock content;
  final String updateKind;  // "agent_message_chunk", etc.
  AcpContentDelta(this.content, this.updateKind);
}
```

Text content becomes `AcpContentDelta(TextContent(text: "..."), "agent_message_chunk")`.

### 2. Parse all content types in agent_message_chunk handler

Replace the text-only check:
```dart
// Before:
if (contentType != 'text') return AcpNonContentUpdate(kind);

// After:
final block = ContentBlock.fromJson(content);
return AcpContentDelta(block, kind);
```

### 3. Parse tool_call_update content array

`tool_call_update` has `content: ToolCallContent[]`. Each item can be:
- `{ type: "content", content: ContentBlock }` → unwrap and parse inner block
- `{ type: "diff", path, oldText, newText }` → `DiffContent`
- `{ type: "terminal", terminalId }` → `TerminalContent`

Emit a list of `ContentBlock` for the tool call update.

### 4. Update RelayChatService to handle AcpContentDelta

`RelayChatService._handleSessionUpdate()` currently handles `AcpTextDelta` → `RelayContentDelta`. Update to handle `AcpContentDelta` → route to conversation state with the full `ContentBlock`.

### 5. Update downstream consumers

Any code that switches on `AcpTextDelta` must handle `AcpContentDelta`. Text content still works — it's just wrapped in `TextContent`.

### 6. Tests (`apps/mobile/test/services/relay/acp_update_parser_test.dart`)

- Parse `agent_message_chunk` with text content → `AcpContentDelta(TextContent(...))`
- Parse `agent_message_chunk` with image content → `AcpContentDelta(ImageContent(...))`
- Parse `agent_message_chunk` with audio content → `AcpContentDelta(AudioContent(...))`
- Parse `agent_message_chunk` with resource content → `AcpContentDelta(ResourceContent(...))`
- Parse `agent_message_chunk` with resource_link → `AcpContentDelta(ResourceLinkContent(...))`
- Parse `agent_message_chunk` with unknown type → `AcpContentDelta(RawContent(...))`
- Parse `tool_call_update` with content array containing diff → `DiffContent`
- Parse `tool_call_update` with content array containing wrapped ContentBlock
- Existing text-mode tests still pass
