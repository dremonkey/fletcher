# Task 30.18: PDF Renderer

**Epic:** 30 — ACP Content Blocks
**Status:** [ ]
**Depends on:** T30.07, T30.08
**Blocks:** none

## Goal

Build a renderer for PDF content arriving as ACP resource content blocks with `mimeType: "application/pdf"`.

## Context

PDFs could arrive as embedded resources (`type: "resource"` with `blob` containing base64 PDF) or as resource links. Rendering requires a PDF viewer package (e.g., `flutter_pdfview`).

## Deferred

**Why deferred:** No ACP agent currently emits PDF content blocks. The resource delivery infrastructure (Epic 31) is also needed for resource_link-based PDFs.

**Revisit when:** ACP agents begin embedding or linking PDF resources in tool_call outputs.

## Relates to

- T30.07: RendererRegistry (would register for `application/pdf`)
- T30.16: ResourceLinkCard (PDF links degrade to metadata card without this renderer)
- Epic 31: Resource Delivery (enables fetching linked PDFs)

## Acceptance criteria

- [ ] PDF resource content renders as an inline viewer or preview
- [ ] Registered in RendererRegistry for `application/pdf`
