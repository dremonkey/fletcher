# Task 30.21: Voice Agent Unsolicited Response TTS

**Epic:** 30 — ACP Content Blocks
**Status:** [ ]
**Depends on:** T30.20
**Blocks:** none

## Goal

Add an unsolicited response listener to the voice agent that feeds relay-originated ACP responses to TTS via `AgentSession.say(stream)`. Enables the "text input + voice output" mode (T30.20).

## Context

When mobile sends a prompt via relay (text mode routing), the relay dual-publishes the response to both `acp` (mobile) and `voice-acp` (voice agent). The voice agent's `RelayChatStream` only processes responses matching an active `requestId`. Unsolicited responses (from relay-originated prompts) are currently ignored.

## Deferred

**Why deferred:** Depends on T30.20 (text input + voice output mode). Without mode 3 in the mobile UI, there's no use case for this listener.

**Revisit when:** T30.20 is implemented.

## Implementation notes (for future reference)

```typescript
// In voice agent setup, add a passive listener on voice-acp:
room.on('dataReceived', (payload, participant, kind, topic) => {
  if (topic !== VOICE_ACP_TOPIC) return;
  const msg = JSON.parse(decoder.decode(payload));

  // Skip responses to our own requests (handled by RelayChatStream)
  if (msg.id && activeRequestIds.has(msg.id)) return;

  // Handle unsolicited session/update with agent_message_chunk
  if (msg.method === 'session/update') {
    const update = msg.params?.update;
    if (update?.sessionUpdate === 'agent_message_chunk') {
      const text = update.content?.text;
      if (text) {
        ttsStream.push(text);  // Feed to ReadableStream → session.say()
      }
    }
  }
});
```
