# Task 30.15: AudioRenderer

**Epic:** 30 — ACP Content Blocks
**Status:** [ ]
**Depends on:** T30.07, T30.08, T30.13
**Blocks:** none

## Goal

Build an `AudioRenderer` widget that displays ACP `audio` content blocks as a metadata card with a play button. Stretch goal: inline playback using `just_audio` or similar. Register with `RendererRegistry` for `audio/*` MIME patterns.

## Context

ACP `audio` content block:
```json
{
  "type": "audio",
  "mimeType": "audio/wav",
  "data": "UklGRiQAAABXQVZFZm10IBAAAAABAAEAQB8AAAB..."
}
```

Audio content is less common than images in typical ACP interactions but may appear in:
- Voice recordings attached to context
- Audio analysis results
- TTS-generated clips

## Implementation

### 1. AudioRenderer widget (`apps/mobile/lib/widgets/renderers/audio_renderer.dart`)

Metadata card showing:
- Audio icon
- MIME type (e.g., "audio/wav")
- Size (computed from base64 data length × 0.75)
- Play button (if inline playback implemented)

```dart
class AudioRenderer extends StatelessWidget {
  final AudioContent block;
  const AudioRenderer({required this.block});
}
```

### 2. Inline playback (stretch)

If time permits, decode base64 → write to temp file → play with `just_audio`:
```dart
final bytes = base64Decode(block.data);
final tempFile = File('${tempDir.path}/audio_${hashCode}.wav');
await tempFile.writeAsBytes(bytes);
await _player.setFilePath(tempFile.path);
```

If `just_audio` is not already a dependency, the metadata-only card is acceptable for v1.

### 3. Register with RendererRegistry

```dart
registry.register('audio/*', (block) => AudioRenderer(block: block as AudioContent));
```

### 4. Tests

- Audio content block renders metadata card
- Size calculation from base64 length
- MIME type displayed

## Not in scope
