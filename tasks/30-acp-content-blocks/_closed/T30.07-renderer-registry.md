# Task 30.07: RendererRegistry with MIME Dispatch

**Epic:** 30 — ACP Content Blocks
**Status:** [x]
**Depends on:** T30.05
**Blocks:** T30.09, T30.14, T30.15, T30.16

## Goal

Build a `RendererRegistry` that dispatches `ContentBlock` instances to type-specific renderer widgets using MIME-pattern matching. Replaces the `_ArtifactContent` switch in `artifact_viewer.dart`. Adding a new content type renderer becomes a single registration.

## Context

**Current dispatch** at `artifact_viewer.dart:399-422`:
```dart
switch (artifact.artifactType) {
  case ArtifactType.diff:     return _DiffViewer(...);
  case ArtifactType.markdown: return _MarkdownViewer(...);
  case ArtifactType.code:
  case ArtifactType.file:     return _CodeViewer(...);
  ...
}
```

Closed enum — every new type requires editing the switch.

**New design:** The `RendererRegistry` maps MIME patterns to widget factories. ContentBlock carries a `mimeType` (or one is inferred from the sealed class type). The registry finds the best match:

```
Registry entries (ordered by specificity):
  text/x-diff   → DiffRenderer
  text/markdown  → MarkdownRenderer
  text/x-*       → CodeRenderer (syntax highlighting)
  text/*         → TextRenderer
  image/*        → ImageRenderer
  audio/*        → AudioRenderer
  resource_link  → ResourceLinkCard
  */*            → RawJsonRenderer (fallback)
```

Pattern matching rules:
1. Exact match wins: `text/x-diff` matches before `text/*`
2. Wildcard subtype: `text/*` matches any `text/...`
3. Global fallback: `*/*` catches everything
4. Special types (DiffContent, ResourceLinkContent) dispatch by sealed class type, not MIME

## Implementation

### 1. Define RendererRegistry (`apps/mobile/lib/widgets/renderer_registry.dart`)

New file:

```dart
typedef RendererFactory = Widget Function(ContentBlock block);

class RendererRegistry {
  // Singleton or injectable
  static final RendererRegistry instance = RendererRegistry._();
  RendererRegistry._();

  final List<_RendererEntry> _entries = [];

  void register(String pattern, RendererFactory factory);
  Widget build(ContentBlock block);
}
```

The `build()` method:
1. For `DiffContent` → always use DiffRenderer (structural dispatch, not MIME)
2. For `TerminalContent` → TerminalCard (or placeholder)
3. For `RawContent` → RawJsonRenderer
4. For others → match `mimeType` against registered patterns
5. No match → RawJsonRenderer fallback

### 2. Pattern matching logic

Keep it simple — no regex. Priority-ordered list:
- Exact string match (e.g., `text/markdown`)
- Prefix match with wildcard (e.g., `text/*` matches `text/plain`, `text/html`)
- Global wildcard `*/*`

Register in specificity order (most specific first).

### 3. Default registrations

Register the built-in renderers at app startup or in the registry constructor:
- `text/x-diff` → DiffRenderer
- `text/markdown` → MarkdownRenderer
- `text/*` → TextRenderer (plain text / code)
- `image/*` → placeholder (Phase 4: T30.14)
- `audio/*` → placeholder (Phase 4: T30.15)
- `resource_link` → placeholder (Phase 4: T30.16)
- `*/*` → RawJsonRenderer

Phase 4 tasks will replace placeholders with real renderers.

### 4. Tests (`apps/mobile/test/widgets/renderer_registry_test.dart`)

- Exact MIME match returns correct renderer
- Wildcard subtype match works (`text/*` matches `text/html`)
- Most specific pattern wins (`text/markdown` over `text/*`)
- DiffContent dispatches by type, not MIME
- Unknown MIME falls through to `*/*` fallback
- No match at all returns RawJsonRenderer
