# Task 30.14: ImageRenderer

**Epic:** 30 — ACP Content Blocks
**Status:** [ ]
**Depends on:** T30.07, T30.08, T30.13
**Blocks:** none

## Goal

Build an `ImageRenderer` widget that displays ACP `image` content blocks. Decode base64 image data in an isolate, enforce a 5MB size cap, and display with loading/error states. Register with `RendererRegistry` for `image/*` MIME patterns.

## Context

ACP `image` content block:
```json
{
  "type": "image",
  "mimeType": "image/png",
  "data": "iVBORw0KGgoAAAANSUhEUgAAAAEAAAAB..."
}
```

The `data` field is base64-encoded. For a typical screenshot (~500KB), the base64 string is ~667KB. This arrives via the relay's chunking mechanism (T30.13).

**Performance concern:** Base64 decoding large strings on the UI thread causes jank. Use `compute()` (Flutter's isolate helper) for the decode.

**Memory concern:** Decoded `Uint8List` images can accumulate. The conversation may have many tool_call_update images (e.g., screenshots from each step). Consider a memory budget or LRU cache.

## Implementation

### 1. ImageRenderer widget (`apps/mobile/lib/widgets/renderers/image_renderer.dart`)

```dart
class ImageRenderer extends StatefulWidget {
  final ImageContent block;
  const ImageRenderer({required this.block});
}
```

States:
- **Loading:** Spinner while base64 decodes
- **Display:** `Image.memory(bytes)` with `BoxFit.contain`
- **Error:** Error message (decode failure, size exceeded)
- **Oversized:** "Image too large (X MB, limit 5 MB)" message

### 2. Isolate-based decode

```dart
Future<Uint8List> _decodeInIsolate(String base64Data) {
  return compute(_decode, base64Data);
}

static Uint8List _decode(String data) {
  final bytes = base64Decode(data);
  if (bytes.length > 5 * 1024 * 1024) throw ImageTooLargeException(bytes.length);
  return bytes;
}
```

### 3. Register with RendererRegistry

```dart
registry.register('image/*', (block) => ImageRenderer(block: block as ImageContent));
```

### 4. Tests (`apps/mobile/test/widgets/renderers/image_renderer_test.dart`)

- Small valid PNG base64 → renders Image.memory
- Invalid base64 → error state
- Oversized image (>5MB decoded) → rejection message
- Loading state shown during decode
- mimeType: "image/jpeg", "image/png", "image/gif" all handled
