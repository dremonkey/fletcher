# Task 30.09: Port Renderers to ContentBlock Input

**Epic:** 30 — ACP Content Blocks
**Status:** [ ]
**Depends on:** T30.05, T30.07
**Blocks:** T30.10

## Goal

Port the existing artifact renderers (Diff, Code, Markdown, Error) from `ArtifactEvent` input to `ContentBlock` input, and register them with the `RendererRegistry`. The `SearchResultsViewer` is dropped — search results arrive as plain text and render in the default text/code renderer.

## Context

**Current renderers** in `artifact_viewer.dart`:
- `_DiffViewer` (lines 429-470) — reads `artifact.file`, `artifact.diff`
- `_CodeViewer` (lines 516-610) — reads `artifact.language`, `artifact.content`, `artifact.file`, `artifact.startLine`
- `_MarkdownViewer` (lines 614-694) — reads `artifact.content`
- `_SearchResultsViewer` (lines 698-792) — reads `artifact.query`, `artifact.results`
- `_ErrorViewer` (lines 796-854) — reads `artifact.message`, `artifact.stack`
- `_RawJsonViewer` (lines 858-900) — reads `artifact.rawJson`

**After:** Each renderer takes a `ContentBlock` subclass as input:
- `DiffRenderer(DiffContent block)` — reads `block.path`, `block.oldText`, `block.newText`
- `CodeRenderer(TextContent block)` or `CodeRenderer(ResourceContent block)` — syntax highlighting
- `MarkdownRenderer(TextContent block)` — reads `block.text` where `block.mimeType == "text/markdown"`
- `ErrorRenderer` — TBD: errors may come as text content or tool_call with `status: "failed"`
- `RawJsonRenderer(RawContent block)` — reads `block.json`

**SearchResultsViewer dropped** (planning decision): search results from ACP agents arrive as plain text in `tool_call_update.content[]`. No special parsing of ripgrep output. The text renders in the default text renderer.

**Design note on diffs:** ACP diffs come from `tool_call_update` with `{ type: "diff", path, oldText, newText }`. This maps to `DiffContent` in the sealed class. The renderer needs to generate a unified diff display from `oldText`/`newText` (the old renderer already does this from `artifact.diff` which was in unified format).

## Implementation

### 1. Create standalone renderer widgets (`apps/mobile/lib/widgets/renderers/`)

Extract renderers from the monolithic `artifact_viewer.dart` into individual files:
- `diff_renderer.dart` — accepts `DiffContent`
- `code_renderer.dart` — accepts `TextContent` or `ResourceContent`
- `markdown_renderer.dart` — accepts `TextContent`
- `raw_json_renderer.dart` — accepts `RawContent`

Preserve the existing rendering logic — just change the input type. Each renderer extracts fields from its `ContentBlock` subclass instead of `ArtifactEvent`.

### 2. Register with RendererRegistry

In the registry initialization (from T30.07):
```dart
registry.register('text/x-diff', (block) => DiffRenderer(block: block as DiffContent));
registry.register('text/markdown', (block) => MarkdownRenderer(block: block as TextContent));
registry.register('text/*', (block) => CodeRenderer(block: block));
```

DiffContent dispatches structurally (by sealed class type, not MIME).

### 3. Handle diff format conversion

Old: `artifact.diff` was a pre-formatted unified diff string.
New: `DiffContent` has `oldText` and `newText`. The renderer needs to compute the diff display. Use a simple line-by-line diff algorithm or the existing diff display logic.

### 4. Error rendering

Tool call failures (`tool_call_update` with `status: "failed"`) may include error content. Route through the text renderer — no special `ErrorRenderer` needed unless the error has structured fields.

### 5. Tests

- Each renderer renders correctly with ContentBlock input
- DiffRenderer handles oldText/newText diff generation
- MarkdownRenderer renders markdown text
- CodeRenderer handles text with syntax highlighting hints
- RawJsonRenderer displays formatted JSON
